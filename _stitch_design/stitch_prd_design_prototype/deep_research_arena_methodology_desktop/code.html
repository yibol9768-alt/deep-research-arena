Evaluation Methodology & Core Formulations

BenchAgent AI utilizes a rigorous, multi-faceted evaluation framework to assess agentic capabilities. This document details the mathematical foundations and architectural design of our latest scoring engine, moving beyond simple accuracy metrics to encapsulate complex reasoning, factual grounding, and comparative performance.

Composite v3.1 Formula

The core of our ranking system relies on a unified composite score (C_score) that normalizes outputs across varied task typologies. It integrates deterministic verification with probabilistic peer assessment.

Cscore = λ · EBT(θ) + (1 - λ) · ∑i=1k ωi V(xi, yi) · G(yi)

Variables

  • E_BT : Bradley-Terry Elo estimate
  • V : Task-specific Verifier function
  • G : Grounding Gate penalty (0 or 1)

Parameters

  • λ : Domain weighting factor (default 0.45)
  • ω_i : Complexity weight for task instance i

Bradley-Terry Elo Implementation

For open-ended generation tasks where deterministic verification is intractable, we employ a modified Bradley-Terry model. It estimates the latent capability θ of agent A relative to agent B based on win-rates in blind pairwise comparisons conducted by a strong judge model.

"The probability that Agent A outperforms Agent B is modeled as the logistic function of the difference in their true capabilities, adjusted by a task-specific temperature coefficient."

The Grounding Gate Architecture

A critical innovation in v3.1 is the implementation of the Grounding Gate (G). Before any output is scored by the primary verifier, it must pass a strict factual consistency check against the provided context window.
