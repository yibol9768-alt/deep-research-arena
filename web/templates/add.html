Add a Deep-Research framework

DR-Arena auto-discovers any runner module that follows the contract below. Drop one Python file in scripts/runners/, refresh the leaderboard build, and your framework joins the Elo board on the next score pass.

The runner contract

# scripts/runners/myframework_runner.py

AGENT_NAME = "my-framework"          # the leaderboard ID. stable forever.

async def run(
    intent: str,                     # the natural-language research task
    model: str,                      # backbone model id (e.g. "qwen3.5-35b-a3b")
    shim_url: str,                   # Tavily-shaped search shim base URL
    proxy_url: str,                  # OpenAI-compatible LLM endpoint
    *,
    timeout_s: int = 1800,           # optional, ignored on smoke test
) -> str:
    """Return the agent's final markdown report (with cited URLs)."""
    ...

Three rules the registry checks

  1. AGENT_NAME must be a non-empty string. This becomes part of the score-file path (data/results/deep_v3/<AGENT_NAME>__<task>_matrix.score.json) — renaming it strands every past run.
  2. run() must be defined with async def. The dispatcher always awaits it.
  3. The first four positional args must be (intent, model, shim_url, proxy_url) in that exact order. Extra kwargs (timeout, debug flags) are fine.

Smoke-test it before opening a PR

# LM Studio must be running with qwen3.5-35b-a3b loaded
python scripts/smoke_test_drs.py --agents my-framework --timeout 90

# Or hit the smoke endpoint from this UI
curl -X POST http://localhost:8000/api/smoke

What "good" looks like

  • ✅ Tier 1 (import + contract) passes — registry doesn't list you in import_errors.
  • ✅ Tier 3 (end-to-end) returns a markdown report with at least 80 chars and a few cited URLs.
  • ⚠️ Reports under 600 chars with reach=0 and judge=0 are filtered as degenerate by the leaderboard builder — your agent will appear in the audit-trail tab instead of the Elo table. That's by design: silent placeholders shouldn't earn rating points.

A note on framework venvs

Heavy frameworks (STORM needs dspy, LDR needs langgraph, ii-researcher needs its full bundle) traditionally live in dedicated venvs on the westd box. The auto-discovery layer is happy with that — your runner can shell out to a subprocess Python in its own venv, exactly like scripts/runners/deerflow_runner.py does. The contract is what the framework integrator sees; what happens behind it is your call.
