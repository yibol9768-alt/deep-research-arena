Why this benchmark

Truthful answers, not fluent fabrications

The three design choices that make this leaderboard different from "ask GPT to evaluate, average the score."

Reachability gate

Every cited URL is HTTP-probed. If the agent fabricates URLs, the entire score is multiplied by zero. Truth is non-negotiable.

composite = reach · quality

Sealed sandbox

No agent touches the real internet. Magento, Postmill, and Kiwix give us a frozen, scrapable corpus — so the golden answer set is reproducible forever.

3 sites · 2,000+ pages · 30 cross-site tasks

Pairwise Elo

Composite scores are noisy and task-correlated, so we don't average them. We pit each agent against every other on every task — exactly like chess Elo — and bootstrap a 95% CI.

Bradley-Terry · N=1000 bootstrap

{# ──────────────────── SANDBOX ──────────────────── #}
The sandbox

Three sites. One sealed corpus.

No agent ever touches the real internet. Every URL it cites must resolve inside this Docker stack — that's how the reachability gate kills fluent-but-fabricated reports.

M
Magento
localhost:7770

~2,000 frozen e-commerce product pages. Specs, reviews, price history, related items.

Shopping & specifications
P
Postmill
localhost:9999

Reddit-style forum threads. Opinion, user experience, edge-case warnings — the kind of context the spec sheets miss.

Community discussion
K
Kiwix (Wikipedia)
localhost:8090

Offline Wikipedia snapshot. Background knowledge, definitions, citations for the agent to ground its synthesis.

Encyclopedic knowledge

Each cross-site task requires ≥120 cited URLs spanning all three sources. Search calls route through a Tavily-compatible shim that only returns sandbox URLs.

{# ──────────────────── METHOD (4-step) ──────────────────── #}
Method

From prompt to Elo, in four steps

{# Connecting line behind step dots #}
1

Run

Each DR framework gets the same one-paragraph prompt and writes a markdown report. Search is routed through a sandbox shim — no live internet.

2

Score

A deterministic scorer produces 7 sub-scores: URL coverage, reachability, quote match, judge pass-rate, citation alignment, depth, presentation.

3

Battle

For every task two agents both scored, the higher composite_v2 wins. The reach gate × quality combine means fabricators forfeit.

4

Elo

Bradley-Terry fits one Elo per agent over all battles. 1000 bootstrap iterations give a 95% CI you can argue about.

Open · reproducible

Run the benchmark on your machine.

Six parallel workers, ~3 hours wall-clock for a fresh 397-pair run. The scorer is deterministic; the golden pool is frozen; the proxy + shim are docker-compose-up away. Or just submit one (agent, task) pair as a PR — it's the cheapest way to engage.

Contribute or reproduce Star on GitHub
Quickstart
# 1. clone + bring up the sandbox
git clone {{ github_url }}.git
cd deep-research-arena
docker compose -f sandbox/docker-compose.yml up -d

# 2. run one (agent × task) pair
.venv-camel/bin/python scripts/run_deep_task.py \
    --agent smolagents --task dr_cross_deep_0001 \
    --backbone deepseek-v4-flash

# 3. score + rebuild the leaderboard
scripts/score_deep_answer.py ...
