How we evaluate

Four steps from a single research prompt to a final Elo. The reachability gate kills any agent that fabricates URLs; pairwise comparisons cancel per-task difficulty noise — same trick that makes chess Elo work.

1

Run

Each framework gets the same prompt, writes a markdown report. Search is routed through a sandbox shim — no live internet.

2

Score

A deterministic scorer produces 7 sub-scores: URL coverage, reachability, quote match, judge pass, citation alignment, depth, presentation.

3

Battle

For every task two agents both scored, the higher composite_v2 wins. Reachability gate × quality means fabricators forfeit.

4

Elo

Bradley-Terry fits one Elo per agent over all battles. 1000 bootstrap iterations give a 95% CI you can argue about.

The sandbox

No agent ever touches the real internet. Every URL it cites must resolve inside this Docker stack — that's how the reachability gate kills fluent-but-fabricated reports. Each cross-site task requires ≥120 cited URLs spanning all three sources.

Frequently asked questions

Common questions about the leaderboard, the sandbox, and how scoring works.

What does "composite_v2" measure exactly?
composite_v2 = reachability · qm_factor · nli_factor · (0.40·url_coverage + 0.40·checklist + 0.20·spec)

The first three terms are multiplicative gates — if reachability is zero (the agent fabricated URLs), the whole score is zero, no matter how fluent the prose. The quality part inside parens is a weighted mix of three deterministic signals.

Why pairwise Elo instead of averaging the composite?

Composite scores are noisy and task-difficulty-correlated. If task #5 is hard, every agent scores low on it — averaging is misleading. Pairwise comparisons cancel out the per-task baseline (the same trick that makes chess Elo work).

How is the golden URL pool generated?

For each task, a scraper walks the three sandbox sites and collects 120-130 must-cite URLs. The pool is frozen — committed to data/golden/deep/. URL coverage is the agent's recall against this set.

What LLM judges the reports?

DeepSeek V4-flash with thinking disabled, served through a local proxy on localhost:8088. Same model, same temperature (0), for every report — judging is fully deterministic.

Can I run this on my own machine?

Yes. docker compose up brings up the sandbox; six parallel workers complete a fresh 397-pair run in ~3 hours. See /contribute for the full recipe.

How do I add a new DR framework?

Drop a runner module at scripts/runners/<your_agent>_runner.py exposing an async def run(intent, model, shim_url, proxy_url) -> str coroutine that returns the markdown report. The registry auto-discovers it. Full walkthrough on /contribute.
