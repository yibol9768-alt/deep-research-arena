Three recipes · pick one

Add a DR framework, or reproduce the leaderboard

Three things you can do here. Each is a separate, self-contained recipe.

1. Run one pair → submit a score 2. Add a new DR framework 3. Reproduce the whole leaderboard
Recipe 1

Run one (agent, task) pair and submit the score

The fastest way to engage with the benchmark. Pick an existing agent, run it on one task, score the output, open a PR with the resulting JSON. No new code required.

1
Clone the repo and bring up the sandbox
git clone {{ github_url }}.git
cd deep-research-arena
docker compose -f sandbox/docker-compose.yml up -d
# starts magento (7770), postmill (9999), kiwix (8090), search shim (8081)
2
Set up the main venv (used by the scorer + most in-process agents)
python -m venv .venv-camel
source .venv-camel/bin/activate
pip install -r requirements.txt
3
Pick an agent + task. Easiest: smolagents on task 0001.
DS_PROXY_URL=http://localhost:8088/v1 \
SHIM_URL=http://localhost:8081 \
.venv-camel/bin/python scripts/run_deep_task.py \
    --agent smolagents \
    --task dr_cross_deep_0001 \
    --backbone deepseek-v4-flash \
    --out-suffix matrix
# writes data/results/deep/smolagents__dr_cross_deep_0001_matrix.md
4
Score the report
JUDGE_BASE_URL=http://localhost:8088/v1 \
JUDGE_MODEL=deepseek-v4-flash \
JUDGE_PROVIDER=openai \
.venv-camel/bin/python scripts/score_deep_answer.py \
    --task dr_cross_deep_0001 \
    --answer data/results/deep/smolagents__dr_cross_deep_0001_matrix.md \
    --out   data/results/deep_v3/smolagents__dr_cross_deep_0001_matrix.score.json
5
Rebuild the leaderboard locally to confirm
.venv-camel/bin/python scripts/build_deep_leaderboard.py
# regenerates data/results/deep_v3/LEADERBOARD_DEEP.md and leaderboard_deep.json
6
Open a PR with both files
  • · data/results/deep/<agent>__<task>_matrix.md
  • · data/results/deep_v3/<agent>__<task>_matrix.score.json

CI re-scores your .md from scratch and checks the JSON matches. If the numbers reproduce, the PR merges and the agent's Elo updates.

Recipe 2

Add a new DR framework

A runner is a single Python module that exposes async def run(intent, model, shim_url, proxy_url) -> str. The string it returns is the markdown report — that's all.

1
Decide: in-process or subprocess?

If your framework can run in .venv-camel alongside everything else, write an in-process runner (fastest). If it needs its own venv (conflicting deps), write a subprocess runner that shells out to its own Python.

2
Drop the runner module at scripts/runners/<your_agent>_runner.py
"""scripts/runners/myagent_runner.py"""
AGENT_NAME = "myagent"

async def run(intent: str, model: str, shim_url: str, proxy_url: str) -> str:
    # Configure your framework so it:
    #   - sends LLM calls to proxy_url (an OpenAI-compatible endpoint)
    #   - sends search calls to shim_url (a Tavily-compatible endpoint)
    # Then run the agent and return the markdown report.
    from myagent import Researcher
    r = Researcher(llm_url=proxy_url, search_url=f"{shim_url}/search")
    return await r.research(intent)

The auto-discovery in scripts/runners/registry.py picks up any *_runner.py with an AGENT_NAME constant — no manual registration.

3
For subprocess agents, add a dedicated venv
python -m venv .venv-myagent
.venv-myagent/bin/pip install your-framework
# Reference it from the runner:
#   subprocess.run(['.venv-myagent/bin/python', driver_script], ...)

See scripts/runners/deerflow_runner.py for a reference subprocess runner. It uses scripts/runners/_runner_lock.py to serialize parallel workers around its shared driver-script path.

4
Smoke-test it on one task
.venv-camel/bin/python scripts/run_deep_task.py \
    --agent myagent --task dr_cross_deep_0001 \
    --backbone deepseek-v4-flash --out-suffix matrix
5
Open a PR with the runner + a few sample score JSONs

CI verifies the runner module is importable and that scoring reproduces. Once merged, your agent gets included in the next full bulk run.

Recipe 3

Reproduce the whole leaderboard

Six parallel workers, deepseek-v4-flash backbone, ~3 hours wall-clock for a fresh ~456-pair run. Each worker gets a shuffled chunk of the queue; per-agent runner locks keep subprocess agents from racing on shared driver-script paths.

1
Bring up the proxy + shim
# OpenAI-compat proxy → DeepSeek (auto-injects thinking:disabled)
OPENAI_PROXY_UPSTREAM=https://api.deepseek.com \
OPENAI_PROXY_KEY=$DEEPSEEK_API_KEY \
.venv-camel/bin/uvicorn integrations.ds_proxy.app:app --host 0.0.0.0 --port 8088 &

# Tavily-compatible search shim that proxies into the sandbox
.venv-camel/bin/uvicorn integrations.search_shim.app:app --host 0.0.0.0 --port 8081 &
2
Generate the queue (every agent × every task)
.venv-camel/bin/python scripts/plan_full_leaderboard.py \
    --agents deerflow flowsearcher-ds gpt-researcher ii-researcher \
             langchain-odr ldr qx-agents smolagents storm \
    --task-range 1-57 \
    --out data/results/deep_v3/run_queue.tsv
3
Launch six parallel workers
bash scripts/parallel_bulk_launch.sh 6
# the launcher splits the queue round-robin (after shuffling) into 6 chunks
# and starts 6 setsid'd workers, each writing its own queue_wN.{log,progress,errors}

Resumable: pairs whose score JSON already exists are skipped. Watch progress with ls data/results/deep_v3/*_matrix.score.json | wc -l.

4
After the run finishes, rebuild the leaderboard
.venv-camel/bin/python scripts/build_deep_leaderboard.py
# writes data/results/deep_v3/LEADERBOARD_DEEP.md
#        data/results/deep_v3/leaderboard_deep.json
5
Optional cleanup pass for transient judge errors
.venv-camel/bin/python scripts/rerun_failed_judges.py --apply
# finds any score JSON with judge_error in checklist / analysis_depth / presentation
# and re-runs only those judges, then recomputes the composites.

Stuck? Open an issue on GitHub.
