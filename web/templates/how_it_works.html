How the leaderboard is produced

A short, plain-English walkthrough — from raw task to final Elo. The short version: every agent answers the same questions on the same data, a deterministic scorer compares their answers to a hidden URL/quote pool, and pairwise battles between agents become Bradley-Terry Elo.

Step 1

The agents run on a sealed sandbox

No agent ever touches the real internet. The sandbox is three local services that look like Amazon, Reddit, and Wikipedia:

  • Magento at localhost:7770 — about 2,000 product pages.
  • Postmill at localhost:9999 — Reddit-style forum threads.
  • Kiwix at localhost:8090 — offline Wikipedia.

Each task is a one-paragraph research prompt that requires citing at least 120 URLs across all three sources. The agent gets the prompt, nothing else, and writes a markdown report.

Step 2

Every report is scored on 7 pillars

The scorer reads the markdown report and produces seven sub-scores. Five are deterministic; two are LLM-judged with strict rubrics.

URL coverage
Did the agent cite the URLs that should be cited? Compares the agent's URLs against a hidden golden pool of must-cite + expected-pool URLs per task.
URL reachability
Do the cited URLs actually resolve in the sandbox? Each URL is HTTP-probed. Hallucinated or fabricated URLs return 4xx/5xx.
Quote match
When the agent quotes a page, does the page actually contain that text? Fuzzy match against the fetched content.
Claim NLI
Are the agent's factual claims entailed by the cited page (rather than contradicted or unrelated)? Natural-language inference per claim.
Checklist (LLM judge)
A 21-item task-specific checklist evaluated by an LLM judge. Each item is a yes/no fact the report must contain.
Citation alignment (ALCE-style)
For each cited URL, does the linked page actually support the surrounding claim? Per-citation precision/recall.
Analysis depth + presentation
Two LLM-judge rubrics for whether the report goes beyond restating sources (analysis) and whether it's structured/readable (presentation).
Spec compliance
Hard-rule checks on word count, citation count, and paragraph count. Pass/fail per dimension; final spec score is the pass fraction.
Step 3

The pillars combine into composite_v2

The headline number on the leaderboard is composite_v2: 

composite_v2 = reachability  ·  qm_factor  ·  nli_factor  ·  ( 0.40·url_coverage + 0.40·checklist + 0.20·spec )

The key idea is the multiplicative gate. URL reachability, quote match, and claim NLI all multiply the quality score. If the agent fabricates URLs (reachability=0), the whole score is zero — no matter how fluent the prose. If the agent quotes pages it never actually fetched (quote match=0), same thing. Truthfulness is non-negotiable.

Without that gate (call it composite_v1), agents that produce confident-but-fabricated 30k-character essays look great. With the gate, the ranking inverts. We chose the gated version on purpose.

Step 4

Pairwise battles → Bradley-Terry Elo

For every task where two agents both have a valid score, we record a "battle": whichever has the higher composite_v2 wins. We fit a Bradley-Terry model — the same model used for chess Elo — to the matrix of battle outcomes. This gives every agent a single Elo number plus a 95% bootstrap confidence interval.

Why pairwise instead of just averaging the composite? Because composite scores are noisy and task-difficulty-correlated. Pairwise comparisons cancel out the per-task baseline, which is the same trick that makes chess Elo work.

Filter

Bad runs are dropped, not zeroed

Some runs are unscoreable: the agent crashes, returns a 50-character error placeholder, or every LLM judge call hits an API error. We filter those out of Elo entirely (they're listed on the audit page) rather than counting them as zeros. A scoring artifact shouldn't be confused with a real loss.

Reproducibility

Deterministic scorer, frozen golden pool

The scorer is deterministic given a task config and a markdown report (the LLM-judge calls use temperature 0). The golden URL pool was scraped once and frozen. Anyone can re-run the leaderboard from the repo and get the same numbers — see the contribute page.
